Prework Study Guide

HTML

  • HTML is the bones / structure of the web page
  • The doc type is declaritive and not case sensitve
  • The head element contains information about the webpage and does not display
  • The title is important for SEO
  • adding alt helps SEO and makes web page more accesible
  • The body element represents the visible content shown to the user
  • elements within elements are referred to as "chilren" of the "parent" element

CSS

  • Inline CSS: style directly in HTML file - quick and permanent changes but less flexible bc must be edited individually.
  • Internal CSS: contained in the head section using classes and id - can edit multiple elements and only applies to that page.
  • External CSS: separate file that is linked in the head section. Preferred in most cases bc it makes changes easier.
  • Selector: defines the element/attribute being mod.
  • Declarations: two components - css property:value
  • CSS Box Model: Every HTML element is a box - from inside out content to padding to border to margin.
  • The * symbol is a selector that reps all elements
  • DRY = dont repeat yourself code
  • Class Attribute: all us to share a CSS rule to any element we choose

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: downloads copy of the main branch from repo
  • git check out main: switches to main branch
  • git add -A: stages your changes in git / terminal
  • git commit -m "enter commit notes": commits the changes to remote repo
